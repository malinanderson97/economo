### 3. Wrap the static HTML titles/subtitles/legends — WITHOUT destroying child markup

These are static HTML in the document body, NOT generated by JS. Two acceptable approaches; pick the one that keeps existing markup intact:

**Preferred — a one-time Boot-region pass** that walks the in-scope containers and wraps ONLY their text nodes:
- Select the target nodes by a scoping hook. Add a class (e.g. `data-wrapsym`) to each in-scope element (`<h3>`, `.sub`, and the `.legend-row span` text) OR select structurally (`.chart-card h3, .chart-card .sub, .legend-row span`).
- For each, run its text through `wrapSymbols` and reassign — BUT the legend spans contain child elements (`.legend-swatch`, `.drill-trigger` with `onclick`). You MUST NOT `innerHTML = wrapSymbols(span.innerHTML)` blindly, because:
  (a) re-`innerHTML` of a node containing an `onclick` attribute is fine (attributes survive in the HTML string), BUT
  (b) `wrapSymbols` splits on `/(<[^>]*>)/g` and skips tags, so child markup is preserved — HOWEVER the `.drill-trigger` text ("derivation ▸") and swatch are inside, and `wrapSymbols` will scan the non-tag text including "derivation" (no symbol match — safe) and the equation text like "Y = f(G,T,r,ε)" (matches — wanted).
  → Acceptable IF done by setting `el.innerHTML = wrapSymbols(el.innerHTML)` on each target, because `wrapSymbols` is tag-aware. CONFIRM by test that the `onclick` drill-triggers still fire after wrapping (browser check).
- Run this pass ONCE at Boot, after the static DOM exists, before/independent of `render()`. It does not need to re-run on render (static text).

> CRITICAL: the legend equation snippets (`IS: Y = f(G,T,r,ε)`, `π = πᵉ + α(Y−Yₙ)/Yₙ + z`, `E = Eᵉ(1+i)/(1+i*)`) are symbol-dense and SAFE — wrap them. The model-name prefixes ("IS:", "LM:", "UIP:") are curve names; with SYMBOL_DEFS-only matching they won't tooltip (fine). Do NOT route legend text through CURVE_DEFS unless step 4 is taken.
